# 翻訳の基本方針

- 原則 gbbinfo3.0 の翻訳に従う
- 口語・カジュアルな原文は維持する（例: `rule_thanks_p2`「マジで疲れた」）
- 固有名詞・ブランド表記は原文維持（Swissbeatbox, GBB, SHOWCASE など）
- 原文が英数字のみの場合は翻訳対象にしない。ハードコードしてよい
- 原文に英単語が含まれる場合、Beatbox 業界固有の単語であるときのみプレースホルダにして、あとで文字列を代入する

例：

```json
{
  "rule_eligibility_7tosmoke": "{SevenToSmoke}は優勝者のみ対象"
}
```

# 技術スタックとファイル構成

- **inlang + Paraglide JS**（`@inlang/paraglide-js`）
- 翻訳ファイル: `messages/{locale}.json`
- 生成コード: `paraglide/messages.js`（`dev` / `build` 時に Vite プラグインが自動生成）
- **baseLocale**: `ja`（`messages/ja.json` が原文の正）
- 対応言語の定義元: `src/constants/languageLabels.ts`（`languageLabels` / `baseLocale` / `ogLocales`）
- `npm run sync:locales` で `languageLabels.ts` の内容を `project.inlang/settings.json` に反映する（sitemap の i18n もこの settings を参照）
- VS Code 拡張 `inlang.vs-code-extension` を推奨

## 呼び出し方

```tsx
import * as m from "../../../paraglide/messages.js";
import { WILDCARD, SWISSBEATBOX } from "~/constants/i18nTerms.js";

m.rule_toc();
m.rule_intro({ Wildcard: WILDCARD, year: String(year), Swissbeatbox: SWISSBEATBOX });
```

## 業界用語定数

プレースホルダに渡す Beatbox 業界用語は `src/constants/i18nTerms.ts` に集約する。各 TSX / Astro ファイルで同名の定数を再定義しない。

| 定数 | 値 | 主なプレースホルダ |
|---|---|---|
| `WILDCARD` | Wildcard | `{Wildcard}` |
| `SOLO` | Solo | `{Solo}` |
| `TAG_TEAM` | Tag Team | `{TagTeam}` |
| `CREW` | Crew | `{Crew}` |
| `PRODUCER` | Producer | `{Producer}` |
| `LOOPSTATION` | Loopstation | `{Loopstation}` |
| `GBB` | GBB | `{GBB}` |
| `SHOWCASE` | SHOWCASE | `{SHOWCASE}` |
| `SHOWCASE_WORD` | showcase | `{SHOWCASE}`（複合語内） |
| `SWISSBEATBOX` | Swissbeatbox | `{Swissbeatbox}` |
| `RC505` | RC505 | `{RC505}` |
| `SEVEN_TO_SMOKE` | 7toSmoke | `{SevenToSmoke}` |
| `BEATBOX` | Beatbox | `{Beatbox}` |
| `BEATBOXER` | Beatboxer | `{Beatboxer}` |
| `YOUTUBE` | YouTube | `{YouTube}` |
| `CHATGPT` | ChatGPT | `{ChatGPT}` |
| `TAG_TEAM_LOOPSTATION` | Tag Team Loopstation | `{TagTeamLoopstation}` |
| `COMEBACK_WILDCARD` | COMEBACK Wildcard | `{ComebackWildcard}` |
| `SOLO_WILDCARD` | Solo Wildcard | `{SoloWildcard}` |

`SHOWCASE` と `SHOWCASE_WORD` は表記が異なる。審査員の `{SHOWCASE}` 告知などは `SHOWCASE`、タイムテーブルの `{SHOWCASE}予選` や `{Producer} showcase` の `showcase` 部分は `SHOWCASE_WORD` を使う。

会場名・年度別 URL・出演者名など、ページ固有の英数字固定文字列はこのファイルに含めない（ハードコード可）。

# キー命名規則

`{ページ/機能}_{説明}` 形式でプレフィックスを付ける。

| プレフィックス | 用途 |
|---|---|
| `rule_` | ルールページ |
| `participant_` | 出場者詳細 |
| `timetable_` | タイムテーブル |
| `wildcard_` | Wildcard 関連 |
| `top7tosmoke_` | 7 to Smoke |
| `others_` | 翻訳説明ページなど |

- キー名に年度（4桁）を入れない。年度は `{year}` / `{prevYear}` / `{nextYear}` プレースホルダかページコンテキストで表現する
- 同一文言・再利用可能なキーは統合する（例: `rule_sbx_judges_historical_note`、`ticket_venue_voltahalle`）
- 既存キーと衝突する場合は意味のある接尾辞で区別する（例: `timetable_solo_final` vs `timetable_solo_final_only`）

# プレースホルダの種類

## 業界用語（PascalCase）

`src/constants/i18nTerms.ts` の定数を `m.xxx()` に渡す。

`{Wildcard}`, `{SevenToSmoke}`, `{Swissbeatbox}`, `{Solo}`, `{Loopstation}`, `{Producer}`, `{SHOWCASE}`, `{Beatboxer}`, `{YouTube}`, `{ChatGPT}`, `{GBB}`, `{Crew}`, `{TagTeam}`, `{RC505}`, `{ComebackWildcard}`, `{SoloWildcard}`, `{TagTeamLoopstation}` など

```tsx
import { WILDCARD, SWISSBEATBOX } from "~/constants/i18nTerms.js";

m.rule_intro({ Wildcard: WILDCARD, year: String(year), Swissbeatbox: SWISSBEATBOX });
```

## 動的値（camelCase）

実行時に代入する。

`{year}`, `{prevYear}`, `{nextYear}`, `{rank}`, `{name}`, `{date}`, `{m}`, `{s}`

# ハードコードしてよい例

- 英数字のみの文字列（翻訳対象外）
- `GBB 2019` のような固定表記（`{GBB}` プレースホルダは語順が言語ごとに変わる箇所のみ使用）
- 部門名テーブル内の `"Solo"`, `"Tag Team"`, `"Loopstation"` など
- 会場名（例: `EX THEATER ROPPONGI`）
- 時刻・タイムゾーン表記（例: `(23:59 CET)`）
  - ただし、日付はl10nが必要なため、必要に応じて翻訳対象とする
- `- For Beatboxers -`, `TEAM MEMBERS` のような英語固定 UI
- URL、メールアドレス（例: `SUPPORT@swissbeatbox.com`）

# 翻訳しない／慎重に扱うもの

- DB 由来の動的データ（出場者名など）
- 国名（`Country.names` JSONB 側で多言語対応）
- ページタイトル（`HeroImage` が担当。`src/views/` 内では `h1` を使わない）

# 新規キー追加の手順

1. `messages/ja.json` にキーを追加する
2. `en.json` / `ko.json` / `de.json` に同じキーを追加する（4 ファイルでキー数を揃える）
3. `npm run dev` または `npm run build` で Paraglide が再生成される
4. TSX で `m.xxx()` を呼ぶ

言語を追加するときは `src/constants/languageLabels.ts` を編集し、`messages/{新locale}.json` を作成してから `npm run sync:locales` を実行する。

# 段階的移行の注意

`JapaneseOnlyPageNotice` 付きの others / travel ページは翻訳対象外（日本語原文のみ）として明示している。それ以外のページでキーを追加したら、同コミットで `m.xxx()` への置換まで行う。
